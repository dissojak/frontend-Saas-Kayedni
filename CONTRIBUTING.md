## What Is Allowed

External contributors may:
- Review the code
- Suggest improvements
- Submit contribution proposals through approved channels

This review/contribution workflow does not transfer ownership and does not grant any license to use the codebase.

## Contribution Process

1. Open an issue describing the proposed change.
2. Wait for maintainer approval before opening a pull request.
3. Keep changes minimal and scoped to the approved issue.
4. Add tests or verification notes when relevant.
5. Ensure no secrets, credentials, or private data are included.

## Contribution Acceptance

- Maintainers may accept, request changes, or reject any contribution.
- Any accepted contribution becomes the exclusive property of StoonProduction unless a separate signed agreement states otherwise.
